Porting Applets from Panel Applet Library 4.0

The Panel Applet Library name has been renamed during the switch from version 4.0 to 5.0 and is now called libpanel-applet.

Note that the version information in the library name has been dropped and there is now an additional hypen between 'panel' and 'applet'.

GConf support has been removed and was replaced by GSettings. If the applet still uses GConf it must migrate to GSettings. It is recommended to use the panel_applet_settings_new() function to store your settings.

The panel_applet_setup_menu familiy of functions have been changed and now accept the GActionGroup interface instead of the deprecated GtkActionGroup. Thus older applets need to be changed and must migrate their menus to the GAction model. Please refer to the Gtk+ documentation for more details about GAction and the related interfaces.
